# Adaptive Protocol

Use this file when the model needs help calibrating **how much structure** a decision deserves.

This is guidance, not a sequence that must be followed literally.

## What Good Behavior Looks Like

- Frame the actual choice instead of answering the surface phrasing.
- Spend more effort when irreversibility, downside, or fragility matter.
- Spend less effort when the case is reversible and the user values speed.
- Mark the assumptions that could flip the answer.
- Choose only the models that materially improve the judgment.
- Gather only the information worth the delay.
- Present the answer in the form that best helps the user decide.

## Depth Heuristic

### Light

Use when:
- stakes are moderate
- the case is reversible
- the user wants speed

Typical style:
- light questioning
- compact answer
- one main answer shape

### Medium

Use when:
- stakes matter
- some key facts are missing
- the decision mixes two domains

Typical style:
- targeted follow-ups
- explicit assumptions
- direct take plus uncertainty

### Deep

Use when:
- downside is serious
- leverage or irreversibility is present
- the answer depends on fragile assumptions

Typical style:
- slower and more careful
- stronger separation of structural vs specialist judgment
- more verification
- stronger survival and optionality framing

### Anchors

Use these as reference points:

- **Light:** choosing between restaurants, scheduling decisions, most software tooling choices, low-cost experiments
- **Medium:** hiring, pricing strategy, product direction, vendor partnerships, 6-18 month commitments
- **Deep:** co-founder or key hire with equity, funding rounds, exit decisions, career pivots with 2+ year lock-in, leveraged bets, irreversible personal commitments

When reversibility and stakes disagree (time-compressed but irreversible), default up to the next depth level.

## Never Forget

- structure first, flourish never
- subtraction beats addition surprisingly often
- the right answer may be wait, no, smaller, slower, or don’t know yet
