# Decision Views

Use this file when the answer shape matters. The analysis core and the presentation view should remain separable.

These are available views, not a required menu.

## Default View

Use for most cases:

- **direct take** — state your actual recommendation or lean
- **compared with what** — name the real alternative, including doing nothing
- **why** — the decisive reason, not all reasons
- **failure mode** — the most likely way this goes wrong
- **uncertainty** — what you don't know that could flip the answer
- **next move** — one concrete action the user should take

## Executive View

Use when the user wants brevity:

- 2–4 sentence answer
- decision, core reason, main caution

## Matrix View

Use when multiple options are genuinely live:

- options × dimensions
- only include dimensions that matter

Good dimensions:
- upside
- downside
- reversibility
- quality
- incentive alignment
- time to learn

## Timeline View

Use when the answer changes by horizon:

- now
- next 6–12 months
- longer-term compounding outcome

## Stakeholder View

Use when different actors bear different costs:

- who gets upside
- who eats downside
- who controls the choice

## Uncertainty Frontier

Use when one or two assumptions dominate:

- stable assumptions
- provisional assumptions
- swing assumptions

## Optionality View

Use when the value of waiting, staging, or sizing down is central:

- what remains open if you act now
- what remains open if you wait
- what closes permanently

## Rule

Do not force one output shape on every problem. Pick the view that best helps the user decide, or combine a small number of views if that is genuinely more useful.
